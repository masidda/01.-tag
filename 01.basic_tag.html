시멘틱태그는 html구조를 설계하는데 있어 태그의 의미를 부여함으로써,
사이트의 구조를 파악하기 용이할 수 있도록 도와주기 위한 태그입니다.

  • header : 머릿글, 제목, 로고, nav 또는 gnb가 있는 영역
  • nav : 네비게이션, 목차 리스트 등 다른 페이지로 이동을 위한 링크영역 표현
  • section : 말 그대로 주제, 카테고리별로 섹션을 구분하는 용도의 태그로 사용, 같은 테마를 가진 여러개의 콘텐츠를 그룹화
  • article : 기사, 블로그, 등 텍스트위주의 페이지를 구성할 때와 외부페이지를 삽입할때 사용
  • aside : 좌측이나 우측 사이드 위치 공간을 의미하며, 본문외의 부수적 내용을 주로 표현
  • footer : 바닥글, 문서 하단에 들어가는 정보 구분 공간을 표현하는 태그
폼양식태그
  • action : 사용자의 입력 데이터를 처리할 URL 주소지정
  • method : 사용자 데이터를 넘겨주는 방식을 지정하는데 get(데이터를 프로그램 환경변수로 넘김), post(표준입력사용) 두가지 방식이 있다.
  • fieldset : form의 양식을 그룹화하여 묶어주는 태그로 테두리가 셍성된다.
  • legend : fieldset의 이름을 지정하는 태그 fieldset 테두리 윗쪽에 생성된다.
